# [Variable関数へのオーバーロードメソッド追加] 改修設計仕様書

## 1. 改修の目的・背景
- **背景**: [Variableインスタンス同士の演算時の記述量が多くより直感的な計算を行いたいと考えたため]
- **目的**: [Variableインスタンスを格納した変数同士で加算・乗算を行うことで利便性の向上を図る]

## 2. 変更内容一覧

| クラス名 | 関数/メソッド名 | 変更の概要 |
| :--- | :--- | :--- |
| `Variable` | `__mul__` | mul関数のオーバーロードを行うメソッドの追加 |
| `Variable` | `__add__` | add関数のオーバーロードを行うメソッドの追加 |

## 3. 詳細設計（Before & After）

### 3.1 [Variable]
#### 変更前 (Before)
- 加算・乗算を行う度にmulメソッドやaddメソッドを呼んでいた

#### 変更後 (After)
- **抽出ロジック**: `__mul__` と `__add__` を定義
- **引数**: `self` と `other` を定義
- **戻り地**: `return add(self, other)`、`return mul(self, other)` を返す

## 4. 影響範囲と整合性
この改修によって修正が必要になる関連箇所。
- **関連1**: `Variable`クラスに `__mul__` と `__add__` を実装する必要がある。
