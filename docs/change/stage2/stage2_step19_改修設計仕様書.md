# [Variable関数への補助機能追加] 改修設計仕様書

## 1. 改修の目的・背景
- **背景**: [Variableクラスでデータを扱う際の利便性を向上したいと考えたため]
- **目的1**: [shapeやdtaypeなど格納した値についての情報を取り出せるようにする]
- **目的2**: [変数にエイリアスをつける機能を追加]

## 2. 変更内容一覧

| クラス名 | 関数/メソッド名 | 変更の概要 |
| :--- | :--- | :--- |
| `Variable` | `shape` | self.data.shapeを返すメソッドを追加。@propertyを関数開始前に付ける |
| `Variable` | `dtype` | self.data.dtypeを返すメソッドを追加。@propertyを関数開始前に付ける |
| `Variable` | `size` | self.data.sizeを返すメソッドを追加。@propertyを関数開始前に付ける |
| `Variable` | `ndim` | self.data.ndimを返すメソッドを追加。@propertyを関数開始前に付ける |
| `Variable` | `__init__` | 引数に`name=None`を追加 |
| `Variable` | `__len__` | インスタンス変数に格納された値のデータの長さを返す |
| `Variable` | `__pepr__` | インスタンス変数に格納された値(self.data)を文字列に変換し返す |

## 3. 詳細設計（Before & After）

### 3.1 [Variable]
#### 変更前 (Before)
- [Variableインスタンスに格納した変数ごとの値を直接取り出せなかった]
- [エイリアスがないため行列計算時における変数ごとの役割が分かりづらかった]

#### 変更後 (After)
- **追加メソッド**: `shape`,`dtype`,`sizs`,`ndim` のメソッドを定義
    - **抽出ロジック**: `x.shape`や`x.ndim` などと記述することで保存された値を取り出す。

- **引数**: `name=None` を引数に変数の名前を定義。
    - **格納形式**: `self.name=name` でインスタンス変数に引数で受け取った名前を保存。
    - **抽出ロジック**: `x.name` と記述することで保存した名前を呼び出す。


## 4. 影響範囲と整合性
この改修によって修正が必要になる関連箇所。
- **関連1**: `Variable`クラスの インスタンスを生成する際にそのインスタンスの名称を指定する必要がある。
